  • RunningEvents: RESTful webapp desenvolvida com o framework Spring Boot

     RunningEvents é uma plataforma criada para promover e ajudar na organização de eventos de corrida. A plataforma possui um sistema de autenticação e autorização, por meio do qual utilizadores, mediante registo e verificação da conta, podem se inscrever em eventos. Administradores podem criar novos eventos, atualizar as informações de eventos existentes e registar os tempos dos atletas em cada estágio da corrida. A partir dos tempos registados na base de dados, a plataforma calcula automaticamente as classificações, permitindo filtrar as classificações por categoria competitiva ou gênero.
     A aplicação foi implementada utilizando o framework Spring Boot. Java é utilizado no back-end para a lógica de serviços, a base de dados foi implementada fazendo uso de PostgreSQL e Java Persistence Architecture. O front-end consiste em HTML, CSS e JavaSript (utiliza-se jQuery para realizar requisições ao servidor). Utiliza-se a API Twilio Sendgrid para enviar emails aos utilizadores, atualmente apenas para verificar suas contas.

  • O Problema dos Mosaicos: solução em Java usando programação dinâmica
     O objetivo deste programa é calcular quantas combinações diferentes de peças podem formar um mesmo desenho em um mosaico. Nesse caso temos 9 tamanhos diferentes de peças (1x1, 1x2, 1x3, 1x4, 1x6, 1x8, 1x10, 1x12, 1x16), que só podem ser usadas horizontalmente. Assim, ao receber a descrição do desenho de um mosaico, o algoritmo deve calcular de quantas maneiras diferentes é possível construí-lo a partir do conjunto de peças disponível.
    Conjunto de peças disponíveis.
    Descrição de um mosaico.

    Cada carácter representa um pixel, letras representam cores (Y = yellow, B = blue, etc.) e pontos representam espaços vazios.


    Existem 16 diferentes combinações de peças que formam o desenho descrito no exemplo acima:
     Diversas técnicas podem ser utilizadas para resolver o problema dos mosaicos. O problema pode ser resolvido por "força bruta", testando todas as combinações possíveis que resultam numa mesma sequência de pixels coloridos. Outra abordagem possível envolve recursão, chamando uma mesma função para partes cada vez menores de uma sequência de pixels coloridos. A solução recursiva, no entanto, implica numa repetição desnecessária de cálculos. Por conseguinte, a solução ótima envolve programação dinâmica, que assim como a solução recursiva, divide sequências de pixels coloridos em partes cada vez menores, mas ao invés de calcular diversas vezes o número de combinações para sequências de um mesmo tamanho, guarda os resultados dos cálculos em um Array, realizando cada cálculo uma única vez.
     Abaixo, segue o código do programa e um plugin que permite executá-lo. Para executar o programa, escreva em Stdin Inputs, na primeira linha, separados por espaços, o número R de linhas e o número C de colunas. As R linhas seguintes devem conter cada uma um número C de caracteres, sendo que pontos representam espaços vazios e letras representam pixels coloridos. Depois, basta clicar em Execute e o resultado aparecerá no terminal.

  • Programa em Java para gestão de fila em paragem de autocarro  Desenvolvido no contexto de uma disciplina sobre Estruturas de Dados e Algoritmos, o objetivo desse projeto é implementar de maneira eficiente uma Queue para a gestão de uma fila numa paragem de autocarro. Um grupo de pessoas chega à paragem em uma determinada hora. Os grupos que chegam posteriormente vão para o fim da fila. O número de pessoas é um atributo do grupo, mas não há qualquer problema de o grupo ter 1 só pessoa. À paragem podem chegar autocarros(só há uma linha), o autocarro tem um certo número de lugares disponíveis e leva todos os passageiros que estejam na fila até acabarem os lugares. Se for necessário um grupo de pessoas é partido ficando no início da fila um grupo de dimensão inferior. Quando o autocarro leva pessoas deve ser calculado o tempo médio de espera das pessoas que embarcam. Ao inicializar o programa, deve-se inserir a data em que serão efetuados os registos, sob o seguinte formato "DD MM AAAA". Posteriormente, para registar a chegada de um grupo de pessoas, deve-se digitar a letra P, seguida da hora, dos minutos e do número de pessoas, separados por espaços, por exemplo: "P 10 00 30" (chegam 30 pessoas às 10 horas). Para registar a chegada de um autocarro, deve-se inserir a letra B, seguida da hora, dos minutos e do número de vagas no autocarro. Para verificar os grupos em espera na fila, deve-se inserir a letra Q. É possível visualizar os arquivos do projeto, compilar e executar o programa por meio da ferramenta JDoodle, clicando aqui. Basta ir até o final da página, selecionar o modo interativo ("interactive"), clicar em "execute" e digitar os comandos no terminal de fundo preto. Se preferir, o projeto também está disponível integralmente neste repositório do Github.
